## PR 2 — Wave component color remap

**Size:** 7 files, ~60–80 line changes (className strings only)  
**Risk:** Zero functional impact — purely visual  
**Prereq:** PR 1 should land first (establishes consistent public palette; lets you see the admin contrast clearly)

### Color substitution table

These are the exact substitutions to make. Apply across all 7 files.

| Old class | New class | Semantic |
|---|---|---|
| `bg-emerald-50` | `bg-leaf/10` | Success/confirmed surface |
| `border-emerald-200` | `border-leaf/30` | Success border |
| `text-emerald-{600,700,900}` | `text-ink` | Text on success surface |
| `bg-green-100` | `bg-leaf/15` | Status badge background |
| `text-green-{600,700,800}` | `text-leaf` | Positive icon/text color |
| `border-green-300` | `border-leaf/30` | Status badge border |
| `bg-blue-50` | `bg-primary/10` | Info callout surface |
| `border-blue-200` | `border-primary/20` | Info callout border |
| `text-blue-{800,900}` | `text-ink` | Text on info surface |
| `bg-blue-600 hover:bg-blue-700` | `bg-primaryDeep hover:bg-primary` | Admin primary action button |
| `bg-amber-{50,100}` | `bg-sun/15` | Warning / mismatch surface |
| `border-amber-{200,300}` | `border-sun/30` | Warning border |
| `text-amber-{700,800,900}` | `text-ink` | Text on warning surface |
| `text-red-700` | `text-destructive` | Error message text |
| `rounded` | `rounded-sm` | Inputs and small chips (min 8px) |

### Files to update (sorted by severity)

#### 🔴 `src/pages/PaymentDashboard.tsx` (25 occurrences)

Key locations:
- `CardHeader className="border-b bg-emerald-50"` → `bg-leaf/10`
- `p className="text-sm text-emerald-900"` → `text-ink`
- `section ... border-emerald-200 bg-emerald-50` → `border-leaf/30 bg-leaf/10`
- `CheckCircle2 className="... text-emerald-600"` → `text-leaf`
- All `border-blue-200 bg-blue-50` info callouts → `border-primary/20 bg-primary/10`
- All `text-blue-{800,900}` → `text-ink`
- `bg-blue-600 hover:bg-blue-700` buttons → `bg-primaryDeep hover:bg-primary`
- `bg-amber-100` icon well → `bg-sun/15`
- `text-amber-{700,800}` + `bg-amber-50 border-amber-200` → `text-ink bg-sun/15 border-sun/30`
- `text-red-700` error → `text-destructive`

The multi-step attestation flow (H1/H2) and "payment sent" state use a mix of all these patterns — remap all systematically.

#### 🟡 `src/pages/admin/AgreementDetailPanel.tsx` (16 occurrences)

Key locations (most added in H3/H5/H8 iterations):
- Direction-arrow icons: `text-green-700` (inbound) / `text-blue-600` (outbound) → `text-leaf` / `text-primary`
- `p className="text-xs text-red-700"` load error → `text-destructive`
- `p className="text-sm text-green-600"` H5 placeholder → `text-leaf`
- `text-amber-{700,900}` + `bg-amber-50 border-amber-300` mismatch banner → `text-ink bg-sun/15 border-sun/30`
- `Badge className="bg-green-100 text-green-800 border-green-300"` uploaded badge → `bg-leaf/15 text-ink border-leaf/30`

#### 🟡 `src/components/admin/DepositRequestDetailPanel.tsx` (13 occurrences)

- `bg-green-600 hover:bg-green-700` (Wave C OperatorReviewForm submit) → `bg-primaryDeep hover:bg-primary`
- Status badges using `bg-green-*` → `bg-leaf/15 text-ink`
- Warning states using `amber-*` → `bg-sun/15 text-ink`

#### 🟡 `src/components/admin/OperatorReviewForm.tsx` (7 occurrences)

Primarily the submit button (`bg-green-600`) and status indicators. Same substitutions as above.

#### 🟠 `src/pages/DepositAgreement.tsx` (4 occurrences)

Minor — status indicators only.

#### 🟠 `src/pages/admin/PickupHandover.tsx` (3 occurrences)

- `Badge className="bg-green-100 text-green-800 border-green-300 text-xs"` → `bg-leaf/15 text-ink border-leaf/30`
- `text-red-700` error state → `text-destructive`

#### 🟠 `src/components/admin/AgreementCommunicationsCard.tsx` (3 occurrences)

Direction arrow icons — same as AgreementDetailPanel.

### What NOT to change

- Do NOT change any logic, guard conditions, or state transitions.
- Do NOT change any admin layout chrome (AdminLayout uses `border-line`, `bg-card`, `bg-muted` — already correct).
- Do NOT add `shadow-sticker` to admin buttons — that's public-CTA-only.
- Do NOT change `text-muted-foreground` — that's a correct shadcn alias.
- The `rounded-lg` and `rounded-md` Tailwind classes are correct (they map to 28px and 18px via the overridden design system). Do not change those.
- Leave `text-destructive` and `bg-destructive` alone — those are already correct system tokens.
